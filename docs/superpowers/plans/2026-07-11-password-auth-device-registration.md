# 密码鉴权与设备自助注册 —— 实现计划

对应设计文档：`docs/superpowers/specs/2026-07-11-password-auth-device-registration-design.md`

> 本计划只锁接口契约和边界，不预写实现代码——具体实现（变量命名、文件组织、
> 测试写法）由 Codex 决定并完成，Claude 负责出方向、审结果。

## 任务 1：服务端 —— 密码鉴权 + 设备清单持久化

**边界**：
- `server/app.py`：`API_KEY` 语义不变（仍是 `X-API-Key` + `hmac.compare_digest()`），
  只是把它当"密码"用；**明文配置项，不引入哈希**。
- 新增 `server/devices.json`（或类似文件）持久化设备清单，替代硬编码 `DEVICES`。
  每条记录字段：`device_id`、`created_at`、`last_active`。**不单独维护显示名**。
- 迁移期需支持**新旧双密码同时有效**（旧 `API_KEY` 环境变量 + 新密码环境变量，
  任一匹配即放行），迁移完成后再由用户手动去掉旧变量——具体开关形式（额外环境
  变量 / 配置项）由 Codex 决定。
- 启动时预置 `win-fukuoka` / `mac-china` 两条设备记录（对应现有硬编码值）。
- JSON 解析失败时**不阻塞启动**，视为空清单并打日志告警。

## 任务 2：服务端 —— 注册 / 列表 / 删除接口

**边界**：
- `POST /api/devices/register`：请求体 `{"device_id": "..."}`，`X-API-Key` 鉴权。
  - `device_id` 规范化：转小写，仅允许 3~32 位字母/数字/连字符，不合规 `400`。
  - 已存在的 `device_id` 视为重连确认，返回 `200`，不新建记录。
  - 未达上限的新设备：写入清单，返回 `200`。
  - 达到上限的新设备：返回 `403` + 明确文案。
  - 注册流程（读取-判上限-写入）必须整体持同一把 `asyncio.Lock`。
- `GET /api/devices`：列出全部设备，合并持久字段与内存在线状态，响应形如
  `[{"device_id":"...","created_at":"...","last_active":"...","online":true}]`。
- `DELETE /api/devices/{device_id}`：删除记录；若该设备当前在线，**必须同时关闭
  其 WebSocket 连接**；删除不存在的设备返回 `404`。删除也走同一把锁。
- `/ws/agent` 只接受**已注册**的 `device_id`，未注册直接 `1008` 拒绝（不再做
  硬编码白名单校验）。
- 设备数上限通过一个可配置项控制（如环境变量 `MAX_DEVICES`），具体默认值和读取
  方式由 Codex 决定。
- WebSocket 断开时更新 `last_active` 同样要经过那把锁，不能绕过。

## 任务 3：Agent 客户端 —— 自助注册 + 设备身份持久化

**边界**：
- `agent/macos/`、`agent/windows/`：启动时先检查本地配置文件里有没有已保存的
  `device_id`。
  - 有 → 直接复用，跳过下面的建议/确认步骤。
  - 没有 → 读取本地 hostname 作为建议名，终端展示，用户回车确认或手动改。
- 确定 `device_id` 后，先调用 `POST /api/devices/register`，成功后**把最终
  `device_id` 写入本地配置持久化**，再用它建立 `/ws/agent` 连接。
- 注册失败（密码错、达到上限等）要给出清晰的终端错误提示，不能吞掉。
- 具体配置文件格式、CLI 交互细节由 Codex 决定，保持跟现有 Agent 代码风格一致。

## 任务 4：前端 —— 设备管理面板

**边界**：
- `server/static/index.html`：新增一个"设备管理"小面板，调用
  `GET /api/devices` 展示列表（设备名 / 最后活跃 / 在线状态），每条一个删除按钮
  （二次确认后调用 `DELETE /api/devices/{device_id}`）。
- 复用页面已有的密码输入框/`X-API-Key` 逻辑，不引入新的鉴权 UI。
- 具体 DOM 结构、样式由 Codex 决定，保持跟现有页面风格一致。

## 任务 5：文档同步

**边界**：
- `docs/protocol.md`：更新鉴权说明（密码替代长 `API_KEY`）、新增注册/列表/删除
  接口说明、设备上限说明、迁移期双密码说明。
- 根 `README.md` 及各 Agent README：更新接入方式为"跑脚本自助注册"，更新凭据
  说明（密码而非长 token）。

## 任务 6：测试

**边界**：
- `server/tests/`：覆盖注册（新增/重连确认/非法格式/达上限）、并发注册不产生
  竞态、`/ws/agent` 拒绝未注册设备、删除（含在线设备被断连、删除不存在返回
  `404`）、`last_active` 只在注册成功/断开时更新、迁移期双密码均可鉴权。
- 具体测试组织方式、fixture 写法由 Codex 决定。

## 验收标准（对应设计文档）

- 密码替代 `API_KEY` 完成鉴权；新旧密码在迁移期都可用。
- 新设备跑安装脚本即可自助注册接入，无需人工预配置；`device_id` 确认后持久化，
  重装/重启不重新生成。
- 达到设备上限时新设备被拒、老设备正常重连。
- 设备管理面板可查看/删除设备，删除在线设备会断开其连接。
- 现有两台设备按迁移路径平滑过渡，不失联。
- 服务端新增测试全部通过；发送流程无回归。
- commit 信息使用中文，不加 `Co-Authored-By` trailer。
